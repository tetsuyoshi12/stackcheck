# NFR要件定義（frontend）

## パフォーマンス
| 要件 | 内容 |
|---|---|
| 初期ロード | Viteのビルド最適化（コード分割・minify）を活用 |
| APIコール | クイズ開始時に1回のみ（5問を一括取得）。回答中はAPIコールなし |
| バンドルサイズ | 外部ライブラリは axios + react-router-dom のみ（最小構成） |

## ユーザビリティ
| 要件 | 内容 |
|---|---|
| ローディング表示 | API取得中はスピナーを表示 |
| エラー表示 | APIエラー時はわかりやすいメッセージを表示 |
| レスポンシブ | モバイル・デスクトップ両対応（Tailwind CSS使用） |
| アクセシビリティ | ボタン・フォームに適切なラベルを付与 |

## セキュリティ
| 要件 | 内容 |
|---|---|
| 管理者認証情報 | sessionStorageに保存しない（メモリのみ） |
| CORS | バックエンドのCORS設定に依存（フロント側での対応不要） |
| 環境変数 | APIのベースURLは `VITE_API_BASE_URL` 環境変数で管理 |

## テスト
| 要件 | 内容 |
|---|---|
| ユニットテスト | Vitest + React Testing Library |
| テスト対象 | 各ページコンポーネントの主要な動作 |
| PBT | スキップ |
