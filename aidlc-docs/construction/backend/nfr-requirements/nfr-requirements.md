# NFR要件定義（backend）

## パフォーマンス

| 要件 | 内容 |
|---|---|
| レスポンスタイム | 500ms以内（Render無料枠のコールドスタート後を除く） |
| AI APIコール | ゼロ（ユーザーセッション中） |
| DBクエリ | 全エンドポイントで単一クエリのみ（N+1なし） |
| コネクションプール | `pool_size=5, max_overflow=0`（Render無料枠PostgreSQLの接続数上限を考慮） |

## 可用性

| 要件 | 内容 |
|---|---|
| SLA | なし（MVP段階） |
| コールドスタート | Render無料枠のため許容（15〜30秒程度） |
| ヘルスチェック | `GET /health` エンドポイントを提供（Renderのヘルスチェック用） |

## セキュリティ

| 要件 | 内容 |
|---|---|
| 管理者認証 | HTTP Basic認証（`secrets.compare_digest`でタイミング攻撃対策） |
| 認証情報 | 環境変数 `ADMIN_USERNAME` / `ADMIN_PASSWORD` から取得 |
| HTTPS | Renderが自動でTLS終端（アプリ側での対応不要） |
| CORS | 許可オリジンを環境変数 `FRONTEND_ORIGIN` で制限 |

## スケーラビリティ

| 要件 | 内容 |
|---|---|
| スケール方針 | v1はRender無料枠の単一インスタンスのみ |
| 水平スケール | v2以降で検討 |

## 保守性

| 要件 | 内容 |
|---|---|
| マイグレーション | Alembicで管理。デプロイ時に自動実行 |
| 環境変数 | `.env`ファイル（ローカル）/ Render環境変数（本番） |
| ログ | uvicornのデフォルトログ（stdout）。Renderが収集 |

## テスト

| 要件 | 内容 |
|---|---|
| ユニットテスト | pytest + httpx（FastAPIのTestClient） |
| テスト対象 | 全エンドポイント（正常系・異常系） |
| PBT | スキップ（シンプルなCRUD構成） |
| カバレッジ目標 | 主要エンドポイントの正常系・異常系を網羅 |
