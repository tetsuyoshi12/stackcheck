# 要件定義書

## インテント分析

- **ユーザーリクエスト**: エンジニア向けスキル確認クイズアプリ。トピックを選んで4択問題に答えることで、知識定着度・スキルの弱点を把握できるサービス。
- **リクエストタイプ**: 新規プロジェクト（Greenfield）
- **スコープ**: フルスタックWebアプリ（バックエンドAPI + フロントエンドUI + 管理画面）
- **複雑度**: Moderate（複数コンポーネント、DB、認証、CORS設定を含む）

---

## 機能要件

### FR-01: トピック一覧表示
- ユーザーはトップ画面でトピック一覧を閲覧できる
- 各トピックにはタイトルが表示される
- `GET /topics` エンドポイントからデータを取得する

### FR-02: クイズ実施
- ユーザーはトピックを選択してクイズを開始できる
- `GET /topics/{id}/questions` で該当トピックの問題5問を取得する
- 問題は登録順（`order` フィールド昇順）で出題する
- 1問ずつ表示し、4択（A/B/C/D）から回答を選択する
- 回答後、即座に正誤判定と解説を表示する
- 「次へ」ボタンで次の問題に進む

### FR-03: セッション結果表示
- 5問終了後に結果画面を表示する（正解数/5）
- セッション内の全問の正誤・解説を振り返れる
- 「別のトピックに挑戦」または「もう一度」ボタンを提供する

### FR-04: セッション管理
- 認証なし・DB保存なし（v1）
- セッションデータはブラウザのセッションストレージのみで保持
- ブラウザを閉じるとリセット

### FR-05: 管理者用トピック登録
- 管理者はWebブラウザからトピックを登録できる
- `POST /admin/topics` エンドポイントを使用
- Basic認証で保護（ユーザー名＋パスワードは環境変数で設定）

### FR-06: 管理者用問題登録
- 管理者はWebブラウザから問題・選択肢・正解・解説を登録できる
- `POST /admin/questions` エンドポイントを使用
- 問題コンテンツはAIツール（Gemini等）で事前生成し、管理画面から手動登録する運用

---

## 非機能要件

### NFR-01: パフォーマンス
- 問題取得はDBから返すのみ（リアルタイムAI生成なし）
- ユーザーセッション中のAPIコール：ゼロ（AI API）
- レスポンスタイム目標：500ms以内（Render無料枠の制約を考慮）

### NFR-02: インフラ・コスト
- バックエンド：Render 無料枠（FastAPI + PostgreSQL）
- フロントエンド：Render 静的サイト配信
- インフラコスト：無料枠で運用（v2移行時に再評価）

### NFR-03: セキュリティ
- 管理APIはBasic認証で保護（ユーザー名＋パスワードは環境変数）
- 本格認証はv2以降
- セキュリティ拡張ルール：スキップ（PoC・MVP段階）

### NFR-04: CORS
- フロントエンドとバックエンドは別オリジン
- FastAPIでCORSミドルウェアを設定（フロントエンドのオリジンを許可）

### NFR-05: テスト
- プロパティベーステスト（PBT）：スキップ（シンプルなCRUD構成のため）
- 基本的なユニットテストとAPIテストを実装

### NFR-06: 可用性
- Render無料枠のため、アイドル時のコールドスタートを許容
- v1はMVP検証優先、SLA要件なし

## データモデル

### Topic（トピック）
| フィールド | 型 | 説明 |
|---|---|---|
| id | int | 主キー（自動採番） |
| title | string | トピック名（例：「Pythonのリスト内包表記」） |
| created_at | datetime | 登録日時 |

**トピック粒度ポリシー**：1トピック = 5問で網羅できる範囲に限定。

### Question（問題）
| フィールド | 型 | 説明 |
|---|---|---|
| id | int | 主キー（自動採番） |
| topic_id | int (FK) | 所属トピック |
| question_text | string | 問題文 |
| option_a | string | 選択肢A |
| option_b | string | 選択肢B |
| option_c | string | 選択肢C |
| option_d | string | 選択肢D |
| correct_option | enum(a/b/c/d) | 正解の選択肢 |
| explanation | text | 解説文（AI事前生成・DB保存済み） |
| order | int | トピック内の出題順（1〜5） |

## APIエンドポイント設計

### ユーザー向け
| メソッド | パス | 説明 |
|---|---|---|
| GET | /topics | トピック一覧取得 |
| GET | /topics/{id}/questions | 該当トピックの問題5問取得 |

### 管理者向け（Basic認証）
| メソッド | パス | 説明 |
|---|---|---|
| POST | /admin/topics | トピック登録 |
| POST | /admin/questions | 問題登録 |

## テクノロジースタック

| レイヤー | 技術 | 備考 |
|---|---|---|
| バックエンド | FastAPI（Python） | 軽量・非同期・自動ドキュメント生成 |
| データベース | PostgreSQL | Render無料枠 |
| ORM | SQLAlchemy + Alembic | マイグレーション管理 |
| フロントエンド | React（TypeScript） | ゼロから実装 |
| フロントエンドビルド | Vite | 高速ビルド |
| デプロイ（BE） | Render（Web Service） | 無料枠 |
| デプロイ（FE） | Render（Static Site） | 無料枠 |

## ユーザーフロー（v1）

```
[トップ画面]
  ↓ トピック一覧を表示
[トピック選択]
  ↓ トピックをクリック
[クイズ開始 - 問題1/5]
  ↓ 選択肢をクリック
[即時フィードバック]
  正解 or 不正解 + 解説表示
  ↓ 「次へ」ボタン
（2〜5問、繰り返し）
  ↓ 5問終了
[結果画面]
  正解数・全問の振り返り表示
  ↓ 「別のトピックに挑戦」or「もう一度」
```

## v1 スコープ

### 含む機能
- トピック一覧表示・選択
- クイズ実施（1問ずつ出題・即時フィードバック・解説表示）
- セッション結果表示
- 管理者用トピック・問題登録画面（Basic認証）

### 含まない機能（v2以降）
- ユーザー認証（Googleログイン）
- ダッシュボード・スキルマップ・継続グラフ
- カテゴリ機能
- 習熟フラグ管理
- 次トピックサジェスト
- ランダム出題オプション
